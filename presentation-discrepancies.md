## КАТЕГОРИЯ 1: Статусы слотов — критическая ошибка

### ❌ Статус `pending` — не существует

**В презентации (слайд "ЛКП Кафедра — черновики, отправка завкафедрой"):**
- Шаг 6: "Кнопка «Отправить на утверждение» меняет статус... Статус: `draft → pending`"
- Слайд "ЛКП Завкафедрой — список": `status=pending` в подвале

**В коде (`packages/types/src/retake.ts`):**
```ts
export type SlotStatus = 'draft' | 'approved' | 'published' | 'closed'
```
`pending` отсутствует. Черновик остаётся `draft` до тех пор, пока зав. кафедрой его не утвердит (`approved`) или не отклонит (остаётся `draft` с `rejectionComment`).

**Что нужно исправить в презентации:**
- Убрать шаги 5 и 6 в flow для dept_worker ("Сводка перед отправкой" и "Отправка на утверждение")
- Слайды `image-26-2.png` и `image-26-3.png` описывают несуществующий workflow
- В слайде завкафедрой заменить `status=pending` → `status=draft`

## КАТЕГОРИЯ 4: Неверное описание BR-03

**В презентации (везде):** "авто-закрытие долга при оценке ≥ 3"

**В коде (`results.service.ts`):**
```ts
const debtStatus = dto.passed ? 'closed' : 'unresolved'
```
Долг закрывается когда `passed = true`. Поле `passed` — отдельный boolean, который преподаватель ставит **независимо** от оценки. Можно поставить оценку 2 и `passed: true` (хотя логически это странно). Поле `grade` может быть null (если только `passed`).

**Как исправить:** "авто-закрытие долга когда преподаватель отмечает «Сдал»" или "при `passed = true`".

## КАТЕГОРИЯ 7: Страницы в проекте, которых нет в презентации

### ⚠️ TeacherSchedulePage для dept_worker (`/schedule/teachers`)

В навигации LKP есть пункт "Расписание преподавателей" для роли `dept_worker`. Это `TeacherSchedulePage.tsx` — позволяет work_worker искать расписание преподавателей для планирования слотов. В презентации этой страницы нет.

### ⚠️ SchedulePage в ЛКС — показана неверно

В ЛКС есть `/schedule` — страница расписания студента. Презентация (слайды 25-26) показывает её как универсальный поиск с параметром `?type=group`. На самом деле она **автоматически** загружает расписание своей группы (через `/mirea-schedule/group/my/planning`), а поиск по другим группам запускается только если группа не привязана (`GROUP_NOT_LINKED`).

### ⚠️ Слайд "Эффект автоматизации" — есть в HTML, нет в slides-data.mjs

В `index.html` между разделами "Технические аспекты" и "Итоги" присутствует слайд `data-screen-label="Эффект автоматизации"` с графиком сравнения AS IS / TO BE. В `tools/slides-data.mjs` этого слайда нет в массиве `SLIDES`. Это значит при перегенерации HTML через `build-deck.mjs` слайд исчезнет.

Слайды показывают поиск по группе, преподавателю и аудитории в LKC. Эндпоинт `/mirea-schedule/search` существует, но это серверный поиск для **системных нужд** (создание слотов). В пользовательском интерфейсе ЛКС показывается только расписание группы студента, не полноценный поиск.
